Análise de Requisitos



Funcionais Não Funcionais Inversos

Funcionais

  • Permitir que alunos consultem a disponibilidade de livros.
  • Exibir prazo de devolução dos livros emprestados.
  • Enviar notificações de lembrete de devolução.
  • Realizar empréstimo de livros online.
  • Permitir que os alunos renovem livros online.
  • Permitir a pesquisa de livros por título, autor ou categoria.
  • Mostrar a lista de livros recomendados com base nos interesses do usuário.
  • Gerar relatórios sobre os livros mais emprestados e menos emprestados.

Não Funcionais

  • O sistema deve ser intuitivo e fácil de usar.
  • O sistema deve ser acessível em dispositivos móveis.
  • Deve suportar múltiplos usuários simultaneamente.
  • O sistema deve ter uma interface amigável e responsiva.
  • O tempo de resposta para as consultas deve ser inferior a 2 segundos.
  • Deve ter uma alta disponibilidade, funcionando 24 horas por dia.
  • O sistema deve ser seguro, protegendo dados dos usuários.
  • Deve ser escalável, permitindo a adição de novos livros sem perda de performance.

Inversos

  • Não permitir que os alunos emprestem livros sem fazer login no sistema.
  • Não permitir o empréstimo de livros com mais de 5 dias de atraso.
  • Não permitir que o sistema mostre livros que não estão disponíveis no momento.
  • Não permitir a exclusão de dados de livros que estejam em uso.
  • Não permitir que alunos vejam informações pessoais de outros alunos.
